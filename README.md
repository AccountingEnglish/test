# 会計英単語アプリ ワイヤーフレーム

要件定義書 v3.1 に基づく、全画面のHTMLモックアップです。

## ファイル構造

```
wireframe/
├── index.html              # ホーム画面（トップ）
├── css/
│   ├── theme.css           # CSS変数（カラー・フォント等）
│   ├── base.css            # リセット・共通レイアウト
│   └── components.css      # 共通コンポーネント
├── js/
│   └── words-data.js       # ダミー単語データ
└── pages/
    ├── range-select.html   # 学習範囲選択
    ├── learn-card.html     # カードめくりモード
    ├── learn-quiz.html     # 4択クイズモード
    ├── learn-input.html    # スペル並べモード
    ├── word-list.html      # 単語一覧
    ├── print-view.html     # 印刷プレビュー
    ├── progress.html       # 進捗詳細
    └── settings.html       # 設定
```

## 動作確認方法

### 方法1: ファイルを直接開く（最も簡単）

`index.html` をブラウザでダブルクリックして開く。
ただし、一部の機能（ローカルストレージ等）はファイルプロトコルでは制限される可能性がある。

### 方法2: 簡易ローカルサーバー（推奨）

ターミナルでwireframeディレクトリに移動し、以下のコマンドを実行。

**Pythonがある場合:**
```bash
python3 -m http.server 8000
```

**Node.jsがある場合:**
```bash
npx serve
```

ブラウザで `http://localhost:8000` を開く。

### 方法3: スマホで確認

PCで上記の方法でローカルサーバーを起動した後、同じWi-Fi上のスマホからPCのIPアドレス（例: `http://192.168.1.5:8000`）にアクセス。
スマホのブラウザの開発者モードで「iPhone 16」サイズに合わせて確認するとより正確。

## 確認すべきポイント

このワイヤーフレームで確認していただきたい事項：

1. **デザインの「AI臭」がしないか**: M PLUS Rounded 1c フォントと早稲田レッドの組み合わせが、想定通りの印象か
2. **画面遷移の自然さ**: ホーム→学習範囲選択→学習画面の流れが、ユーザー負担を感じないか
3. **学習モードの操作感**:
   - カードめくりのタップ操作は直感的か
   - 4択クイズのフィードバック表示は適切か
   - スペル並べの文字パレットの大きさ・配置は使いやすいか
4. **単語一覧の使いやすさ**: 絞り込み・検索・選択・印刷への導線が分かりやすいか
5. **進捗画面の情報量**: 多すぎず少なすぎないか
6. **タップ領域**: ボタンや項目が押しやすいサイズか（44px以上を確保）

## 注意事項

- このワイヤーフレームは**見た目の合意を取るための使い捨て**です
- 本実装時には、IndexedDBによる永続化、適切なJSアーキテクチャ、本番用の単語データ等を伴って書き直します
- アニメーションは最小限。本実装時にはより丁寧な動きを設計します
- 単語データは20語のサンプルのみ

# 会計英単語アプリ 本実装【段階1】

要件定義書 v3.2 に基づく本実装の段階1です。

## 段階1で実装した内容

### データ層
- `data/words.json` - 76語の用語データ（第6回授業資料由来 + AI生成解説・例文）
- `js/storage.js` - IndexedDB抽象化レイヤー（Repository Pattern）
- `js/words.js` - 用語データの読み込み・フィルタ・検索
- `js/progress.js` - 学習進捗管理と判定ロジック（要件定義書 4.6節）
- `js/ui.js` - 共通UIユーティリティ
- `js/word-list.js` - 単語一覧画面のロジック

### 画面
- **ホーム画面（index.html）**: 動的進捗表示・全体統計を実装
- **単語一覧画面（pages/word-list.html）**: 動的データ表示・絞り込み・検索・選択・印刷導線を実装

### 段階1で動作するもの
- ホーム画面で全76語のうち何語習得済みかを表示
- 単語一覧で76語をリスト表示
- カテゴリ・難易度・学習状況でフィルタリング
- 英単語・日本語訳・解説でキーワード検索
- 複数選択して印刷プレビューへ遷移する導線
- IndexedDBへの永続化（学習データはブラウザに保存される）

### まだ動作しないもの（段階2・3で実装予定）
- 学習モード（カードめくり・4択クイズ・スペル並べ）：ワイヤーフレームのまま
- 進捗詳細画面：ワイヤーフレームのまま
- 印刷プレビュー：ワイヤーフレームのまま
- 設定画面のデータリセット・エクスポート機能

## 動作確認方法

### 必須：ローカルサーバーで起動する

ファイルを直接ブラウザで開いても動きません（fetch APIがCORS制限で動かないため）。
必ず以下のいずれかでローカルサーバーを起動してください。

**Pythonの場合（macOS/Linuxはほぼプリインストール済み）:**
```bash
cd app/
python3 -m http.server 8000
```

**Node.jsの場合:**
```bash
cd app/
npx serve
```

その後、ブラウザで `http://localhost:8000` を開く。

### 動作確認の手順

1. ホーム画面を開く
   - 「0 / 76語」と表示される（初期状態）
   - 統計に「0日」「—」「0」が表示される
2. 「単語」タブ（下部）をタップ
   - 76語のリストが表示される
3. フィルタチップで絞り込み
   - 「やさしい」を押すと初級・基礎の用語のみ表示
   - 「習得済み」を押すと該当なしと表示される（まだ学習していないため）
4. 検索ボックスに「現金」と入力
   - "Cash and cash equivalents" が表示される
5. リスト項目をタップして複数選択
   - 下部に赤いバーが現れ、選択数が表示される
   - 「印刷プレビュー」ボタンで遷移できる（ただし印刷ページはまだワイヤーフレーム）

### IndexedDBの確認方法

ブラウザの開発者ツールで確認できます。

**Chrome/Edge:**
1. F12で開発者ツールを開く
2. 「Application」タブ → 「Storage」 → 「IndexedDB」
3. 「AccountingVocabApp」を展開
4. `progress`、`stats`、`meta` の3ストアが作成されている

**Safari:**
1. 開発メニューから「Webインスペクタ」
2. 「ストレージ」タブ → 「Indexed Database」
